# 매직디펜스 — 디자인 문서

- **작성일**: 2026-05-16
- **문서 타입**: 디자인 스펙 (Design Spec)
- **상태**: 초안 (사용자 검토 대기)

---

## 1. 프로젝트 개요

### 1.1 정체성
모바일 안드로이드용 머지(합치기)형 마법 디펜스 게임.
플레이어는 마법사를 무작위 소환하고, 같은 마법사끼리 합쳐 강화해서, 끝없이 몰려오는 적을 막아낸다.

### 1.2 목표
- **결과물**: 안드로이드 폰에서 실제로 플레이 가능한 **프로토타입**.
- **검증 질문**: "이 게임의 머지+레인 디펜스 루프가 재미있는가?"
- **범위 외 (Out of Scope, 프로토타입 단계)**:
  - 결제, 광고, 인앱
  - 멀티플레이 / PvP / 서버
  - 계정·로그인·클라우드 세이브
  - 출시(스토어 등록), 마케팅
  - 일러스트·애니메이션 제작
  - 사운드 (효과음·BGM은 1단계에선 생략하거나 기본 무료 에셋만)

### 1.3 대상
- 연령: 전체이용가
- 디바이스: 안드로이드 모바일 (세로 화면)
- 첫 테스트 환경: PC 브라우저 + 안드로이드 폰 크롬으로 접속

---

## 2. 기술 스택

### 2.1 핵심 결정
- **게임 엔진**: Phaser 3 (HTML5/JavaScript)
- **언어**: JavaScript (ES2022). 프로토타입 단순화 우선, 타입스크립트 도입은 2단계 검토.
- **빌드/번들**: Vite (Phaser 공식 템플릿 사용 가능)
- **저장**: `localStorage` (최고 점수·웨이브 기록)
- **배포 (프로토타입)**: 로컬 dev 서버 → 폰 크롬으로 같은 네트워크 URL 접속
- **추후 APK 포장**: Capacitor (프로토타입 검증 후 별도 단계)

### 2.2 왜 Phaser 3인가
- 셋업 거의 없음 (브라우저만 있으면 실행)
- AI 친화 (JavaScript 코드 생성·디버깅 사이클 빠름)
- 2D 게임 전용 엔진으로 자료 풍부
- 모바일 터치/제스처·물리·스프라이트 기본 지원

## 3. 게임플레이

### 3.1 코어 루프
1. 플레이어가 **소환 버튼**을 탭 → 골드 소비 → **머지보드**의 빈 칸에 **랜덤 클래스의 Lv1 마법사**가 등장
2. 같은 **클래스 + 같은 레벨** 마법사 두 명을 **드래그**해서 겹치면 → **Lv+1**로 합쳐짐
3. 마법사들은 자기 위 **적 레인의 적**을 자동 공격
4. 적이 레인 끝(아래)에 도달하면 → **본진 HP 감소**
5. HP가 0이 되면 게임 오버 → "웨이브 N까지 버팀" 결과 표시 → 최고 기록 갱신

### 3.2 모드
- **무한 웨이브 (Endless Wave)**: 한 가지 모드만. 한 판 시작하면 적이 점점 강해지며 무한히 옴.
- **스테이지 없음, PvP 없음, 미션 없음** (프로토타입 범위).

### 3.3 승리/패배 조건
- **패배(게임 오버)**: 본진 HP가 0 이하
- **승리 조건 없음** — 점수(=도달 웨이브)가 곧 결과

## 4. 마법 클래스 (4종)

### 4.1 정의
| 기호 | 이름 | 역할 | 공격 방식 | 강한 상황 |
|---|---|---|---|---|
| 🔥 | 불 | 단일 딜러 | 가장 가까운 적 1명에게 강한 데미지 | 단일 강적 (보스/엘리트) |
| ❄️ | 얼음 | 디버퍼 | 단일 적에게 데미지 + 이동속도 감속(슬로우) | 빠른 적, 핵심 적 묶기 |
| ⚡ | 전기 | 체인 다중타격 | 1명을 때리고 근처 적으로 번개가 튐 (체인) | 한 줄로 늘어선 적 |
| 🌍 | 땅 | 광역 CC | 한 지점 반경 안 모든 적에 데미지 + 짧은 기절 | 좁은 곳에 뭉친 적 |

### 4.2 레벨업 (머지)
- **머지 규칙**: 같은 클래스 + 같은 레벨 두 명 → Lv+1 한 명. 빈 칸은 그대로 남음.
- **최대 레벨**: Lv5
- **레벨별 강화**: 레벨 1당 데미지 ×2, 공격 속도 +10%. (밸런스 수치는 5장 참고)
- **클래스 변경 불가**: 머지는 같은 클래스끼리만. 다른 클래스끼리는 합쳐지지 않음.

### 4.3 클래스 분포 (랜덤 소환 확률)
- 4종 모두 균등 25% (프로토타입 단순화). 나중에 가중치 조정 가능.

## 5. 화면 구성 (UI/UX)

### 5.1 해상도와 비율
- **기준 해상도**: 720×1280 (9:16 세로)
- **반응형**: `Phaser.Scale.FIT`으로 다양한 폰 해상도에 맞춤

### 5.2 화면 레이아웃

```
┌──────────────────────────────┐
│ ❤️ HP 20    웨이브 3    💰 250 │  ← (1) 상태바 (상단 8%)
├──────────────────────────────┤
│  레인1    레인2    레인3    레인4│
│   👹       👹                   │
│   ↓        ↓        💀          │
│            👹       ↓     👹    │  ← (2) 적 레인 영역 (40%)
│  👹                              │     4개 세로 통로
│                                 │     적은 위→아래로 이동
├──────────────────────────────┤
│  🔥    ❄️    □     🌍           │
│  □     🔥    🌍    ❄️           │  ← (3) 머지 보드 (40%)
│  🌍    □     🔥    🌍           │     4×4 = 16칸
│  □     🌍    ❄️    🔥           │
├──────────────────────────────┤
│   [🧙 소환 50G]    [⏩ 1x/2x]    │  ← (4) 액션바 (12%)
└──────────────────────────────┘
```

### 5.3 영역별 설명

#### (1) 상태바
- **HP**: 본진 체력 (시작 20)
- **웨이브**: 현재 웨이브 번호
- **골드**: 보유 골드

#### (2) 적 레인 영역
- **4개 세로 레인** (left to right: 1, 2, 3, 4)
- 적은 각자 자기 레인의 **맨 위에서 등장**, 일정 속도로 **아래로 이동**
- 적이 레인 맨 아래(머지보드 상단)에 도달 → 본진에 데미지(적 크기/종류 기반) → 적은 사라짐
- 적 종류와 등장 패턴은 6장 참고

#### (3) 머지 보드 (4×4)
- 컬럼(세로 열) 4개 × 행(가로 줄) 4개 = **16칸**
- 각 컬럼은 위쪽의 같은 번호 적 레인과 매칭됨 (예: 보드 1번 컬럼 마법사 → 1번 레인 적만 공격)
- 마법사는 컬럼 내 **위치 무관**하게 같은 컬럼의 적을 공격 (사거리 무한)
- 보드 칸은 빈 칸/마법사 점유 두 가지 상태

#### (4) 액션바
- **🧙 소환 버튼**: 골드 소비해 보드 빈 칸에 랜덤 마법사 1명 생성. 빈 칸 없으면 비활성/오류 표시.
- **⏩ 속도 버튼**: 게임 진행 속도 1x ↔ 2x 토글. 적 이동·공격 속도 모두 영향.

### 5.4 터치 UX
- **소환**: 버튼 탭
- **머지**: 마법사 드래그 → 다른 마법사 위에 드롭
  - 같은 클래스+레벨이면 머지 성공
  - 다르면 원래 자리로 복귀
- **칸 이동**: 마법사 드래그 → 빈 칸에 드롭 (위치 재배치)
- **속도 변경**: 버튼 탭
- **게임 오버 후**: "다시 시작" 버튼

### 5.5 보조 화면
- **타이틀 화면**: 게임 제목 + [시작] 버튼 + 최고 기록 표시
- **게임 오버 화면**: 도달 웨이브, 최고 기록, [다시 시작] 버튼
- **일시정지**: 1단계에선 생략 (속도 0x도 없음)

## 6. 적 시스템

### 6.1 적 종류 (프로토타입 2종)
| 기호 | 이름 | HP | 이동속도 | 본진 데미지 | 비고 |
|---|---|---|---|---|---|
| 👹 | 일반 (Goblin) | 20 | 1.0 | 1 | 기본 적 |
| 💀 | 빠른 (Skeleton) | 10 | 2.0 | 1 | 슬로우 효과 받기 좋음 |

(보스 등 추가 적은 2단계 — 프로토타입 검증 후)

### 6.2 웨이브 시스템
- **웨이브 길이**: 각 웨이브는 정해진 **마릿수**가 모두 스폰되면 끝남 (시간이 아니라 마릿수 기반)
- **웨이브 간격**: 마지막 적 처치 후 5초 휴식 → 다음 웨이브 시작 (이때도 골드는 누적)
- **마릿수**: 웨이브 1 = 10마리, 웨이브마다 +2마리 (W2=12, W3=14, …)
- **스폰 간격**: 웨이브 1 = 1.5초, 웨이브마다 -0.05초 (최소 0.4초)
- **레인 분배**: 적은 4개 레인 중 **랜덤 1개**에 등장 (단, 동일 레인 연속 3마리 초과 금지)
- **난이도 증가 (웨이브당)**:
  - 적 HP: ×1.15
  - 적 종류 비율: 웨이브 5부터 💀(빠른) 30% 등장, 웨이브 10부터 50%

## 7. 경제 / 밸런스 (초안)

| 항목 | 값 | 비고 |
|---|---|---|
| 시작 HP | 20 | |
| 시작 골드 | 100 | 초반 2회 소환 가능 |
| 자동 골드 | +2/초 | 게임 시작 직후부터 |
| 적 처치 보상 | +1 골드/마리 | 일반/빠른 동일 |
| 소환 비용 | 50G (시작) | 소환할 때마다 +5 |

### 7.1 마법사 Lv별 스탯
| 클래스 | Lv1 데미지 | Lv1 공격속도 | 타게팅 | 효과 범위 |
|---|---|---|---|---|
| 🔥 불 | 10 | 1.0/초 | **같은 컬럼**에서 가장 앞선 적 1명 | — |
| ❄️ 얼음 | 5 | 1.5/초 | **같은 컬럼**에서 가장 앞선 적 1명 | 슬로우 30% 2초 |
| ⚡ 전기 | 4 | 1.2/초 | **같은 컬럼**에서 가장 앞선 적 1명 | 첫 타격 후 **인접 컬럼 포함** 가장 가까운 적 2명에게 50% 데미지 체인 |
| 🌍 땅 | 3 | 1.5/초 | **같은 컬럼**에서 가장 앞선 적 1명 | 그 적 주변 1.5칸 반경 내 모든 적(**컬럼 무관**)에 데미지 + 0.5초 기절 |

#### 타게팅 규칙 (모든 클래스 공통)
- 모든 마법사의 **타게팅(첫 타격 대상)은 자기 컬럼 적으로 한정**한다.
- 단, ⚡ 전기와 🌍 땅의 **2차 효과**(체인·광역)는 컬럼 경계를 넘을 수 있다.
- 같은 컬럼에 적이 없으면 마법사는 공격을 쉰다(다른 컬럼 적을 노리지 않음).

- 레벨업당 데미지 ×2, 공격속도 ×1.1

> 이 수치는 **초안**이고, 실제 플레이 테스트하면서 조정해야 함. 구현 단계에서 쉽게 변경 가능하도록 **config 파일/객체로 분리**할 것.

## 8. 시스템 아키텍처 (모듈 구분)

게임 코드를 작은 단위로 쪼개서 각자 책임이 명확하도록 설계.

### 8.1 주요 모듈
| 모듈 | 책임 |
|---|---|
| **Scene: TitleScene** | 타이틀 화면, 시작 버튼, 최고 기록 표시 |
| **Scene: GameScene** | 메인 게임플레이 화면 (상태바·적·보드·액션바) |
| **Scene: GameOverScene** | 게임 오버 결과 화면 |
| **MergeBoard** | 4×4 보드 상태 관리 (칸별 마법사), 드래그·머지 처리 |
| **EnemyLane** | 4개 레인 관리, 적 스폰·이동·도달 처리 |
| **Enemy** | 적 1마리의 상태 (HP, 위치, 효과: 슬로우/기절) |
| **Mage** | 마법사 1명의 상태 (클래스, Lv, 쿨다운) |
| **AttackResolver** | 마법사가 어떤 적을 칠지·데미지 계산·이펙트 |
| **EconomyManager** | 골드 관리, 소환 비용 계산 |
| **WaveManager** | 웨이브 진행, 적 스폰 패턴, 난이도 스케일링 |
| **Config** | 모든 밸런스 수치 (한 곳에 모음) |
| **SaveStore** | localStorage 최고 기록 저장/불러오기 |

### 8.2 의존성 흐름
```
GameScene
  ├── MergeBoard ── Mage[]
  ├── EnemyLane ── Enemy[]
  ├── AttackResolver  (MergeBoard와 EnemyLane을 참조)
  ├── EconomyManager
  ├── WaveManager  (EnemyLane에 적 추가)
  └── (모두 Config 참조)
SaveStore는 TitleScene, GameOverScene에서 사용
```

### 8.3 통신 방식
- 모듈 간 직접 호출은 최소화, **Phaser의 EventEmitter**로 이벤트 기반 통신:
  - `enemy:reached-base` (HP 감소)
  - `enemy:killed` (골드 보상)
  - `mage:merged` (UI 갱신)
  - `wave:started` / `wave:ended`
  - `game:over`

## 9. 에러 처리 / 엣지 케이스

- **빈 칸 없을 때 소환**: 소환 버튼 자동 비활성, 짧은 "보드 가득" 토스트
- **골드 부족**: 소환 버튼 비활성 (회색)
- **드래그하다 같은 자리에 놓음**: 아무 일 없음, 원위치
- **드래그 도중 게임 오버**: 드래그 무효 처리, 게임 오버 화면 전환
- **모든 적이 한 레인에 몰릴 때**: 의도적 동작 (운). 그러나 한 레인에 5마리 이상 동시에 몰리지 않도록 스폰 시 체크.

## 10. 비주얼 (1단계: 이모지+도형)

### 10.1 방침
- 모든 캐릭터는 **이모지**로 표시
  - 마법사: 🧙 (또는 클래스별 색상 원 안에 🔥❄️⚡🌍 작은 아이콘)
  - 적: 👹 (일반), 💀 (빠른)
  - 본진: 🏰
- 보드 칸·HP바·버튼은 **사각형/원** + 색상
- 마법 이펙트: 단색 라인·원 (불=빨강 선, 얼음=하늘색 원, 전기=노란 지그재그, 땅=갈색 원)

### 10.2 2단계 교체 계획 (재미 검증 후)
- Kenney.nl 또는 itch.io의 무료 카툰 디펜스 에셋팩 사용
- Phaser 스프라이트 로더로 이모지 자리에 PNG 교체
- 모듈 구조상 비주얼 레이어만 갈아끼우면 됨 (게임 로직 영향 없음)

## 11. 테스트 전략

### 11.1 자동 테스트
- 게임 로직(머지 규칙·데미지 계산·웨이브 스케일링)은 **순수 함수로 분리**해서 단위 테스트
- Phaser/렌더링과 무관한 코드는 Node에서 Jest/Vitest로 검증

### 11.2 수동 테스트
- 매 코드 변경 후 브라우저 새로고침으로 즉시 플레이
- 폰 크롬으로 같은 URL 접속해 실제 터치 동작 확인 (드래그·머지)
- 최소 한 판을 끝까지 플레이해 게임 오버 흐름 검증

### 11.3 재미 검증 기준
- 한 판 평균 3분 이상 플레이되는가?
- "한 판만 더" 마음이 드는가?
- 4종 클래스가 모두 쓸모 있는가? (한 클래스만 강하면 안 됨)

---

## 12. 성공 기준 (프로토타입 완료 조건)

다음이 모두 만족되면 프로토타입 완성:

1. PC 브라우저에서 타이틀 → 게임 → 게임오버 → 재시작 전체 흐름이 동작
2. 안드로이드 폰 크롬에서도 같은 흐름이 끊김 없이 동작
3. 머지(드래그)가 폰 터치로 정확히 작동
4. 4종 클래스 모두 구현되어 시각·기능적으로 구분됨
5. 무한 웨이브가 점점 어려워지면서 게임 오버 발생
6. 최고 기록(웨이브 수)이 localStorage에 저장·표시됨
7. 1단계 비주얼(이모지+도형)로 모든 요소 표현

---

## 13. 향후 단계 (이 문서 범위 밖)

프로토타입 완성 후, 재미 검증 결과에 따라:

- **재미 있다 → 2단계**: 카툰 에셋 교체 + 사운드 추가 + 보스 적 + 클래스 추가
- **부분적으로 재미 → 조정**: 밸런스·UX 다듬기
- **재미 없다 → 피벗**: 코어 메커닉 변경 (예: 보드 크기 변경, PvP 추가 등)
- **출시 결심**: Capacitor로 APK 포장 → 구글 플레이 콘솔 등록
